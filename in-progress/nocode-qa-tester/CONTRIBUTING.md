# Contributing

This project is in early validation. The most valuable contribution right now is **feedback**, not code.

## How to give feedback

Open an issue with one of these labels:

- `use-case` — describe your current workflow and the QA problem you face
- `test-case` — suggest a test scenario that should be included
- `pricing` — your thoughts on pricing model
- `stack` — preference on tech choices (Node vs Python, etc.)

## When code contributions open

Once the MVP is scoped and validated, a `good first issue` label will appear.
